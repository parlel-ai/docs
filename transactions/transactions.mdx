## Overview

Every action execution and commercial rule use in Parlel creates a transaction record. Transactions are immutable — they can't be edited or deleted. They give you a complete, auditable history of every customer interaction: what was requested, who requested it, what authentication tier was in place, what executed, and what the outcome was.

This matters for compliance, dispute resolution, and operational visibility.

## What gets logged

| Event | Transaction created |
|---|---|
| Action executed | Yes — always |
| Commercial rule fired | Yes — always |
| Authentication tier upgrade | Yes — always |
| Fallback flow triggered | Yes — always |

Every transaction includes:
- Timestamp
- Customer identifier (hashed — no raw PII stored)
- Authentication tier at time of event
- Action or rule involved
- Input data and output result
- Success or failure state
- Reference ID (`#PRL-XXXXXXXX`)

## Customer privacy

Parlel stores customer identifiers as one-way hashes. Raw PII — email addresses, names — is never stored in the transaction log. Customer lookup for transaction history uses the same hashing mechanism.

## Viewing transactions

Go to **Transactions** in the portal. Filter by:
- Date range
- Action type
- Authentication tier
- Success / failure state
- Customer (via hashed identifier lookup)

Each transaction shows full event detail — inputs, outputs, reference ID, and timestamp.

## Reference ID

Every transaction is assigned a unique reference ID in the format `#PRL-XXXXXXXX`. This ID can be shared with customers as a confirmation reference and used to look up the transaction in the portal.
